spar-kit is a lightweight workflow system for building with AI agents. It keeps intent, scope, and validation visible—so you spend less time re-litigating decisions and more time shipping.

What spar-kit gives you

A repeatable way to collaborate with agents and keep changes coherent from idea to shipped result.

Less thrash on requirements Fewer surprises mid-implementation Stronger quality via validation Reusable knowledge after shipping

Why this works

Make intent explicit

Define the change in a crisp spec before you start coding, so the agent can stay aligned.

Turn intent into tasks

Translate the spec into an ordered plan you can check off—no “magic” leaps in the middle.

Retain what you learned

After shipping, update docs and archive the change so future work starts from truth, not memory.

How it works

  1. Specify — clarify the change: summary, problem, goal, scope.
  2. Plan — complete success criteria, then derive ordered, checkable tasks.
  3. Act — implement, validate, and keep decisions consistent with the spec.
  4. Retain — update docs, align the spec with what shipped, and archive the change folder.
Tip: spar-kit is intentionally simple—plain markdown artifacts, minimal ceremony, and a workflow your team can adopt incrementally.
