# Agent instructions

<!-- spar-kit:start -->
Before non-trivial work:
1. Read `.spar-kit/.local/tools.yaml`.
2. Check repo scripts / `justfile`
3. For **implementing or planning** a change — or anything **user-facing**, **multi-file**, or **unclear** — use SPAR starting with `spar.specify`. If unsure, use SPAR. The SPAR workflow defines how changes are processed and documented in this repo. It also defines how the AI and user must collaborate with eachother before coding/implementation.

**Trivial-only exception:** obvious one-file fixes (typos, a single link) with no product ambiguity — full SPAR not required.

When finishing non-trivialwork:
When validation passes and your current objectives are complete, read `spar.retain` to finalize documentation.
